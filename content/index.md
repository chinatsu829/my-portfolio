## My Value

### Core Value
**ラベル**: 複雑な業務課題を構造化し、ユーザーが迷わないUIを設計する

**具体的要素**:
- 大規模マスタ・業務フローの構造化
- 将来の機能追加を見据えたデザインシステム
- 論理的なUIロジックの構築

### 共創

**① PdM / PO との共創**
- テキスト: 要件の可視化
- 役割: 言語化しきれない要件を、素早くプロトタイプで具体化し、認識のズレをゼロにする。

**② Engineer との共創**
- テキスト: 実装を考慮したUI設計
- 役割: 技術的制約やコンポーネントの再利用性を理解し、エンジニアが迷わず実装できるUIデザインを構築する。

### 根拠のあるUI設計

**① Stakeholder / User との対話**
- テキスト: 本質的なニーズの探究
- 役割: 表面的な要望に惑わされず、ヒアリングから真の課題を抽出し、プロダクトの進むべき方向を合意形成する。

**② 論理的なUI検討**
- テキスト: 説明できるUIを作る
- 役割: なぜこのUIを採用したかを論理的に説明できる設計を行う。感覚ではなく根拠のあるデザイン判断を積み重ねる。
